# ferry_llama — flat C wrapper around llama.cpp for PHP FFI

PHP FFI cannot reliably pass llama.cpp's `llama_model_params` / `llama_context_params`
**by value**: the struct layout PHP FFI infers mismatches the Clang-built `llama.dll`, so
`llama_model_load_from_file` crashes (`llama-hparams.cpp` fatal error). See
[`docs/DEBT_REPORT.md`](../../docs/DEBT_REPORT.md) §12.

This wrapper solves it: a tiny MSVC-built DLL that constructs those structs **inside C**
(real ABI) and exposes a **flat API** — only pointers, ints, strings and byte buffers cross
the PHP↔native boundary. Verified end to end on **CPU (~96 tok/s)** and **GPU / RTX 4060
(~250 tok/s)** with `qwen-0.5b.Q4_K_M.gguf`.

## What you need

Put everything in one directory (default `D:\FerryAI`), which must be on `PATH` at runtime:

| Item | Why | Source |
|------|-----|--------|
| `llama.dll`, `ggml.dll`, `ggml-base.dll`, `ggml-cpu-*.dll` | CPU inference engine | github.com/ggml-org/llama.cpp/releases |
| `ggml-cuda.dll` + `cudart64_*.dll`, `cublas64_*.dll`, `cublasLt64_*.dll` | GPU (CUDA) inference | llama.cpp CUDA release (`*-bin-win-cuda-*`) + NVIDIA CUDA Toolkit |
| `llama.h`, `ggml.h`, `ggml-cpu.h`, `ggml-backend.h`, `ggml-alloc.h`, `ggml-opt.h`, `gguf.h` | build the wrapper (same commit as the DLLs) | github.com/ggml-org/llama.cpp (`include/` + `ggml/include/`) |
| `llama.lib`, `ggml.lib` | link the wrapper (auto-generated by `build.ps1` from the DLLs) | generated via `dumpbin`/`lib` |
| A `.gguf` model | weights + tokenizer | huggingface.co (e.g. `bartowski/*-GGUF`) |
| Visual Studio 2022 (cl/lib/dumpbin) | compiler | visualstudio.microsoft.com |

## Build

```powershell
powershell -File native/llama-wrapper/build.ps1 -LlamaDir D:\FerryAI
# -> D:\FerryAI\ferry_llama.dll   (also creates llama.lib / ggml.lib if missing)
```

## Smoke test (CPU + GPU)

```powershell
$env:PATH = "D:\FerryAI;" + $env:PATH
```

## Flat API (FFI cdef)

```c
void  ferry_llama_backend_init(void);
void  ferry_load_backends(const char* dir);   // load ggml-cpu-*/ggml-cuda DLLs from dir
void  ferry_llama_backend_free(void);
int   ferry_supports_gpu_offload(void);
void* ferry_load_model(const char* path, int n_gpu_layers);  // n_gpu_layers=0 CPU, 99 = all on GPU
void  ferry_free_model(void* model);
void* ferry_new_context(void* model, int n_ctx, int n_threads);
void  ferry_free_context(void* ctx);
int   ferry_n_vocab(void* model);
int   ferry_n_embd(void* model);
int   ferry_tokenize(void* model, const char* text, int* out_tokens, int max_tokens, int add_bos);
int   ferry_generate_greedy(void* ctx, void* model, const int* prompt_tokens, int n_prompt,
                            int max_new, char* out, int out_size);
```

## Notes / limits

- `ggml_backend_load_all` etc. are exported by **ggml.dll** (not ggml-base.dll) — link `ggml.lib`.
- Backends must be loaded from the llama directory (`ferry_load_backends($dir)`) because
  llama.cpp resolves them next to the **host exe** (php.exe), not the DLL.
- Greedy sampling only; temperature/top-k/top-p/grammar are TODO in the wrapper.
- Not yet wired into `FerryAI\LlamaBackend\Runtime\NativeLlamaRuntime` (next step; see
  `docs/DEBT_REPORT.md` §12).
- Runs in standalone PHP; under PHPUnit the ggml global constructors conflict with the test
  runner (§12).
