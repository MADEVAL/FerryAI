### 2026-07-04 ‚Äî Steps 42‚Äì45: ONNX FFI layer (runtime seam, tensor, model, backend)

**Dependency decision (documented deviation):** `phpmlkit/onnxruntime` referenced throughout the design
docs **does not exist on Packagist**. The documented fallback (`SKILL.md` ¬ßWhat We Use) ‚Äî
**`ankane/onnxruntime`** (v0.3.0) ‚Äî is used instead as the native ONNX Runtime FFI binding.

**Architecture ‚Äî a single, mockable FFI seam:**
- `Runtime/OnnxSession.php` ‚Äî marker interface for an opaque loaded-session handle.
- `Runtime/OnnxRuntimeInterface.php` ‚Äî the seam. Every value crossing it is plain PHP data, so the
  whole backend is unit-testable without the native library.
- `OnnxTypeMapper.php` ‚Äî ONNX element-type ‚Üí `DType`, provider name ‚Üí `Device`, and device ‚Üí
  ordered provider names (with CPU fallback).
- `OnnxTensor.php` ‚Äî implements `Tensor` as a value wrapper; arithmetic/transform throw
  `BadMethodCallException` (tensor math belongs to the `tensor` package).
- `OnnxModel.php` ‚Äî implements `Model`; normalises inputs (Tensor‚Üíarray), delegates to the seam,
  wraps outputs as `OnnxTensor`; caches input/output metadata; `unload()` makes `run()` throw.
- `OnnxBackend.php` ‚Äî implements `Backend`; maps available providers ‚Üí devices, resolves the target
  device (`Device::resolve`), maps device ‚Üí providers, creates the session, builds `ModelMetadata`.
  Remote sources ‚Üí `ModelLoadException`; missing file ‚Üí `ModelNotFoundException`; unavailable runtime
  ‚Üí `BackendNotAvailableException`.

**Native FFI boundary (excluded from static analysis, covered by integration tests):**
- `OnnxRuntimeFactory.php` ‚Äî the sole coupling point to `ankane/onnxruntime` (availability, version,
  `GetAvailableProviders` via FFI, model creation, graph-optimisation mapping).
- `Runtime/NativeOnnxRuntime.php` ‚Äî implements the seam over the factory; normalises ankane node
  metadata (`tensor(float)` ‚Üí `float`, symbolic dims ‚Üí `-1`).
- `Runtime/NativeOnnxSession.php` ‚Äî wraps ankane `Model`.
- Excluded via `phpstan.neon` (`excludePaths.analyse`) and `psalm.xml` (`ignoreFiles`) because ankane
  is an untyped third-party FFI library; symbols remain resolvable for reflection.

**Test doubles + tests:**
- `tests/Double/MockOnnxRuntime.php` + `tests/Double/MockOnnxSession.php` ‚Äî deterministic seam doubles.
- Unit: `OnnxTensorTest`, `OnnxModelTest`, `OnnxBackendTest`.
- Integration: `tests/Integration/Onnx/OnnxRuntimeIntegrationTest.php` (`@group integration`,
  `@coversNothing`) ‚Äî skips gracefully when the native library is absent.

**Verification:**
- `composer check` ‚Äî cs-check OK, PHPStan lvl8 OK, Psalm lvl3 No errors, PHPUnit 185/185 (404 assertions).
- `php -l` on all three native files ‚Äî no syntax errors.
- Smoke: `new OnnxBackend()->isAvailable()` returns `false` gracefully (no native lib) ‚Äî no fatal.
- `composer test-integration` ‚Äî 3 skipped, 0 failures (FFI boundary behaves as designed).

**Status:** `onnx-backend` Phase 1 complete (10 files + seam/doubles). The `ROCm`/`OpenVINO` providers
remain Phase 4.
